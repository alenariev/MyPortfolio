## Презентация
Мой личный сайт-портфолио, разработанный в качестве тестового задания. Я не стала делать типовой шаблонный сайт, а оформила его в стиле личного кабинета своего основного проекта — NutriTarget (интеллектуальный сервис КБЖУ). Мне нравится работать со сложными цветовыми схемами и выстраивать логичную структуру интерфейса.

## Ссылки проекта
**Живой деплой (Vercel):** https://my-portfolio-ar-iota.vercel.app

## Использованный стек
•   Frontend: HTML5, CSS3 (переменные, кастомная сетка Grid/Flexbox, адаптивная верстка под мобильные устройства), TypeScript, Vite (инструмент сборки).
•   Backend (API): Python (Serverless Functions на платформе Vercel), Flask. Обработчик находится в папке api/contact.py.
•   Интеграция сторонних API: Асинхронное взаимодействие через Fetch API (JSON-формат), Telegram Bot API (для мгновенных уведомлений), OpenAI API (для умных автоответов).

## Реализация формы обратной связи
Форма полностью отвечает требованиям тестового задания и реализует законченный цикл взаимодействия пользователя с сервером.
1. **Валидация на клиенте:**
   Перед отправкой на сервер TypeScript проверяет введённые данные с помощью регулярных выражений:
   • **Имя:** только буквы, длина от 2 символов.
   • **Телефон:** проверяется соответствие стандартным маскам номеров (минимум 10 цифр).
   • **Email:** строгая проверка на корректность адреса почты.
   • **Сообщение:** длина не менее 5 символов.
2. **Интерфейс и состояния (UX):**
   • **Загрузка:** При отправке кнопка блокируется (`disabled`), текст меняется на "Отправка...", включается CSS-лоадер (вращающийся круг).
   • **Успех:** При успешном ответе сервера форма сбрасывается, выводится золотистое сообщение об успешной доставке.
   • **Ошибка:** Если валидация не прошла или сервер вернул ошибку, выводится красное сообщение с подробным описанием проблемы.
3. **Бэкенд-логика (`contact.py`):**
   • Принимает POST-запрос с JSON, валидирует данные на стороне сервера.
   • Интеграция с ИИ: Передает комментарий пользователя в OpenAI API для генерации персонализированного умного автоответа (в случае отсутствия API-ключа бесшовно переключается на качественный статический шаблон).
   • Мгновенные уведомления: Дублирует все данные заявки (Имя, Телефон, Email, Сообщение) в Telegram-чат владельца через Telegram Bot API.
   • Email-рассылка: Отправляет два письма через безопасный протокол SMTP_SSL Яндекса: одно мне с информацией о контакте, второе — копия пользователю с вложенным автоответом от AI для подтверждения отправки.

## Как запустить проект
Для запуска требуется использовать официальную утилиту Vercel CLI.
1. Установите Vercel CLI глобально (если не установлен):
bash
  npm install -g vercel
2. Запустите проект из корневой папки:
bash
  vercel dev
Эта команда автоматически поднимет локальный сервер, объединяющий фронтенд на Vite и Python-функции. Форма будет успешно слать запросы на /api/contact.

## Использование ИИ
В процессе разработки я использовала ИИ в качестве технического ассистента для ускорения рутинных операций.
 Что было сделано с помощью ИИ:
• Быстро сгенерирована базовая конфигурация файлов tsconfig.json и vercel.json под Python-окружение.
• Подготовлен шаблон асинхронной функции fetch для отправки формы в TypeScript.
• Написана стартовая анимация уходящей вверх шторки-загрузчика на CSS Keyframes.
• Частично написана проделанная работа в readme по моим кратким описаниям.

Что пришлось исправлять и дорабатывать вручную:
ИИ хорошо генерирует изолированные куски кода, но часто ошибается в деталях и стилистике. Всю интеграцию и отладку я делала самостоятельно:
• Дизайн и фирменный стиль: ИИ предлагал шаблонную серую цветовую гамму. Я полностью переписала CSS-переменные, внедрила палитру проекта NutriTarget.
• Сетка и адаптивность: Автоматически сгенерированный макет "ломался" на планшетах и мобильных (круглые декоративные элементы наезжали на текст, ломались пропорции рамки с фотографией). Я вручную перестроила сетки на Grid/Flexbox и прописала медиа-запросы.
• Баги отрисовки и поведения: 
• Браузеры при обновлении страницы прокручивали её вниз. Я добавила принудительное управление историей скролла (history.scrollRestoration = 'manual') и скролл наверх при загрузке.
• Фотография с тенью: Настроила стили .photo-frame со смещенной жесткой золотой тенью и свойством object-fit: cover для качественного отображения фотографии без искажения пропорций.

## Безопасность проекта
Все конфиденциальные данные (пароль приложения SMTP Яндекса, токен Telegram-бота, ID чата и API-ключ OpenAI) полностью изолированы от исходного кода. Они не хранятся в репозитории GitHub, а настроены через переменные окружения (Environment Variables) на Vercel.
